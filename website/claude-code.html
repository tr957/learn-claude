Claude Code

When you need Claude to work with your actual files — not just answer questions about them.

What Claude Code is

Normal Claude works in a chat window. You type a message, Claude responds, you copy the output and use it somewhere. Claude has no access to your computer — it can only see what you paste in.

Claude Code is different. It runs in a terminal on your computer (the Terminal app on a Mac) and can read files, write files, make changes across multiple documents, and run tasks end to end — without you having to copy and paste anything. Think of it as the difference between asking someone for advice versus having them sit at your desk and actually do the work.

Claude Code is built for technical tasks — coding, editing files, managing projects. It is not a replacement for normal Claude chat, which handles the vast majority of 1kx's day-to-day use cases. But there are specific situations where it is clearly the better tool.

Chat vs Code — the simple decision

Stay with normal Claude chat when: you are writing, researching, analysing, or thinking through something. If your workflow is "I type a prompt and use the response," chat is the right tool. This covers almost everything on this site.
Consider Claude Code when: you need Claude to make changes to actual files on your computer, the task involves more than two or three documents that would be painful to copy-paste, or you want Claude to run a repeatable process across a whole folder or project rather than a single conversation.

What you can do that normal chat cannot

  • Edit multiple files at once. Change a template across ten documents in a single instruction. Update every instance of a name, format, or section structure in one go.
  • Work across a whole project. Point Claude at a folder of documents and ask it to do something consistently across all of them — reorganise, reformat, find inconsistencies, extract information.
  • Run tasks end to end. Give Claude an instruction and let it complete the full task without you managing each step. It can create files, modify them, check its own work, and report back.
  • Build and update tools. If you or your team have any internal tools, templates, or systems that live in files, Claude Code can help maintain and update them without manual editing.

Real examples from building this site

This website — the one you are reading — was built and maintained using Claude Code. A few concrete examples of what that looked like:

Adding search to every page

The search bar in the top-right navigation needed to be added to all ten pages with identical HTML. In chat, that would have been ten separate edits. In Claude Code, it was one instruction: "add the nav search input to every HTML file in this folder."

Updating the navigation structure

When the advanced section was added to the site, every page needed a new navigation structure with new links. Claude Code updated all pages simultaneously, consistently, in one session.

Building the search index

The search feature on this site works off a file called search-data.js that contains every prompt on the site. Claude Code built that file by reading all the HTML pages and extracting the relevant content — a task that would have taken hours manually.

Three main use cases at 1kx

Outside of technical work, there are three practical scenarios where Claude Code becomes worth considering:

  1. Document management at scale. If you have a large number of documents — notes, reports, templates — that need to be reorganised, reformatted, or searched in a structured way, Claude Code can handle that as a single task.
  2. Maintaining internal templates. If 1kx has shared templates, standard documents, or internal tools that get updated periodically, Claude Code can make changes across all copies simultaneously.
  3. Automating repetitive file work. Any task that involves the same action on many files — renaming, reformatting, extracting specific information — is a good candidate.

What to expect if you try it

Claude Code is accessed through the Terminal app on a Mac. If you have never used Terminal before, the fastest path is to ask someone technical at 1kx to set it up for you the first time — it takes a few minutes and you only do it once.

Once it is running, you use it in plain English. You describe what you want, Claude Code does it, and shows you what it changed. It asks for confirmation before making changes you might want to review first.

It is not for everyone, and it does not need to be. The chat interface covers almost all day-to-day use cases. But if you find yourself repeatedly doing manual work across multiple files, it is worth knowing this option exists.

Not sure if Claude Code applies to your situation? Describe what you're trying to do to someone using this site — if the answer is "this would take 20 minutes of copy-pasting", Claude Code is probably the right tool.
